Backup & Restore System

  • • Full backups include database + all attachments (comments, chat messages) in a ZIP file
  • • DB-only backups include just the database without attachment files
  • • Automatic backups are created every 12 hours (keeps last 10, auto-deleted)
  • • Manual backups are auto-deleted after 20 (oldest removed first)
  • • Download backups to your local machine for external storage and disaster recovery
  • • Upload backups from local machine when server needs to be restored after failure
  • • On startup, database is automatically checked and restored from latest backup if corrupted
  • • Create manual backups before major changes to preserve that state

Important Notes

  • • MANUAL backups (green) are auto-deleted after 20 (oldest removed first)
  • • AUTO backups (blue) are kept for the last 10 automatic backups only
  • • UPLOADED backups (purple) come from your local machine and are kept forever
  • • You can also manually delete any backup using the delete button
  • • Restoring a backup will replace the current database and attachments completely
  • • All data added after the backup was created will be lost
  • • Current database and uploads are saved as "corrupted_[timestamp]" before restore
  • • After restoring, you may need to log in again
  • • Download backups regularly to local storage for disaster recovery
  • • If server crashes completely, restore by: 1) Upload backup, 2) Click restore
💡

Disaster Recovery Workflow

  1. Regular backups: Download full backups (ZIP files) to your local machine regularly
  2. If server fails: Set up new server, install application
  3. Upload backup: Use the upload form above to upload your local backup ZIP file
  4. Restore: Click the restore button on the uploaded backup
  5. Result: All data, tasks, comments, and attachments are restored to the backup state
